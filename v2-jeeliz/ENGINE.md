# v2 ‚Äî Jeeliz VTO engine (current, 2026-07-04)

Sunglasses virtual try-on, WebAR. Successor to v1 (MindAR) ‚Äî switched for fit accuracy, stability, and occlusion.

## Engine
- **Tracking + rendering:** Jeeliz VTO Widget (`lib/JeelizVTOWidget.js`, vendored) ‚Äî neural tracker trained specifically on faces wearing glasses. Auto-scale per face, temple occlusion, light reconstruction (ambient + directional), PBR, raytraced shadows, temporal antialiasing.
- **Models:** streamed at runtime from Jeeliz GlassesDB CDN (`glassesdbcached.jeeliz.com/sku/<sku>` + `appstatic.jeeliz.com`). **Internet required.**
- **No build step.** Static files, any HTTPS server.
- Production users of same engine: Jean Paul Gaultier, Lensway, Pardon!.

## How it was built, step by step
1. Vendored widget JS from `jeeliz/jeelizGlassesVTOWidget` (GitHub, legacy-but-free product).
2. **Did NOT load `lib/JeelizVTOWidget.css`** ‚Äî that file styles their demo page; its fullscreen adjust-notice swallowed all touch input (adjust mode appeared frozen). All styling is ours in `index.html`.
3. Required DOM ids kept: `JeelizVTOWidget`, `JeelizVTOWidgetCanvas`, `JeelizVTOWidgetAdjust`, `JeelizVTOWidgetAdjustNotice`, `JeelizVTOWidgetAdjustExit`, `JeelizVTOWidgetLoading`.
4. API surface used: `JEELIZVTOWIDGET.start({sku, searchImageMask, callbackReady, onError})`, `.load(sku)`, `.enter_adjustMode()`, `.exit_adjustMode()`.
5. 14 Ray-Ban SKUs wired (catalog: `glassesSKU.csv` in their repo, 365 SKUs total). All validated headless.
6. Thumbnails (`thumbs/<sku>.png`) generated by test rig: each SKU rendered on a reference face in headless Chrome (fake-webcam canvas stream), glasses region cropped 160√ó80.
7. Adjust mode: notice restyled as compact bottom card, `pointer-events:none` except Done button; `body.adjusting` hides tray so canvas gets full drag surface. Drag is damped by engine design.

## Possibilities / how to expand
- **More models (minutes):** add `{sku, label}` to `MODELS` in `index.html` from the 365-SKU catalog; regenerate thumbnail via rig.
- **UI ideas:** photo capture button (`JEELIZVTOWIDGET` has pause/resume), LUT color filters (`set_LUT`), product info cards per SKU, share link per model (`?sku=` param ‚Äî small addition).
- **Watch try-on:** Jeeliz does faces only. Watch = separate build: MediaPipe Hands wrist anchor + three.js (assets found: Khronos ChronographWatch, Polyhaven digital_wrist_watch).

## Repo & deployment
- **Repo:** `github.com/nagorik-xr/xr-projects` (public, org-owned ‚Äî separate from any personal namespace).
- **Live:** https://nagorik-xr.github.io/xr-projects/ (gallery) ‚Üí `/v2-jeeliz/` (this demo) + `/v1-mindar/` (archive).
- **Hosting:** GitHub Pages, `main` branch, root path. Deploy = commit + push to `main`, live in ~1 min. No build step, no secrets, no CI.
- **Layout:** repo root = gallery `index.html`; each demo is a self-contained folder (`v1-mindar/`, `v2-jeeliz/`) with its own `ENGINE.md`. Add future demos as new top-level folders + a gallery card.
- Camera requires HTTPS ‚Äî Pages provides it; that's why local-network testing failed (no trusted cert on phone) and tunnels were blocked (network allows only ports 80/443).

## Known limits
- Models require Jeeliz CDN ‚Äî offline demo impossible in v2 (v1 is fully self-hosted).
- Drag adjustments reset on model switch (engine behavior).
- Legacy product: works today, no vendor support guarantee.
